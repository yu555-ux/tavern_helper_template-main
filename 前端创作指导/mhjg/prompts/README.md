# 游戏剧情生成提示词模板

## 文件说明

- `story_template.json`: JSON格式的提示词模板，包含完整的结构定义和说明
- `story_template.txt`: 文本格式的提示词模板，可以直接复制到角色卡的提示词中使用

## 使用方法

### 方法一：直接使用文本模板

1. 打开 `story_template.txt`
2. 复制全部内容
3. 粘贴到角色卡的"额外问候语"或"系统提示词"中
4. 根据需要调整变量引用部分（使用EJS模板语法）

### 方法二：使用JSON模板

1. 打开 `story_template.json`
2. 提取 `template.instruction` 字段的内容
3. 根据需要进行自定义修改
4. 添加到角色卡的提示词中

## 模板结构

模板包含三个主要部分：

### 1. 剧情文本 (<maintext>)

- **字数要求**：500-1300字
- **内容要求**：包含对话、行为、行动、外界变化、玩家决定等
- **格式要求**：可以使用换行和特殊标点符号

### 2. 剧情总结 (<sum>)

- **格式**：时间 | 地点 | 人物 | 事件
- **长度**：1-2句话
- **要求**：精准、简洁

### 3. 变量更新 (<UpdateVariable>)

- **Analysis部分**：列出所有可能变化的变量，用Y/N标记
- **更新语句**：使用MVU系统的更新函数（_.set, _.add, _.assign, _.remove）
- **路径规则**：所有路径必须加上[0]后缀

## 变量更新规则

### 基本更新函数

- `_.set('路径[0]', 旧值, 新值)`: 设置新值
- `_.add('路径[0]', 变化量)`: 增减数值
- `_.assign('数组路径[0]', 新元素)`: 向数组添加元素
- `_.remove('数组路径[0]', 元素)`: 从数组删除元素

### 常见变量路径

- 资源：`资源.金币[0]`、`资源.声望[0]`、`资源.建材[0]`
- 时间：`时间.天数[0]`、`时间.当前时间[0]`
- 旅店：`旅店.营业状态[0]`、`旅店.受欢迎度[0]`、`旅店.入住率[0]`
- 设施：`设施.设施列表[0]`
- 客人：`客人.客人列表[0]`
- 员工：`员工.员工列表[0]`
- 法则：`旅店.旅馆法则.当前法则[0]`

### 数组变量更新注意事项

对于数组类型的变量（设施列表、客人列表、员工列表），更新时需要：

1. **添加元素**：使用 `_.assign('数组路径[0]', 新元素)`
2. **更新元素**：使用 `_.set('数组路径[0]', 旧数组, 新数组)`，需要提供完整的旧数组和新数组
3. **删除元素**：使用 `_.remove('数组路径[0]', 要删除的元素)`

```
<maintext>

店主放下酒杯，仔细打量着这位精灵。从她身上散发出的魔法气息来看，她是一位经验丰富的法师。

经过一番交谈，店主对艾莉丝的印象很好。她不仅专业，而且性格温和，非常适合前台接待的工作。

"那么，你愿意接受这份工作吗？"店主问道，"月薪200金币。"

"当然愿意！"艾莉丝眼中闪烁着兴奋的光芒，"我什么时候可以开始工作？"

"明天就可以，"店主笑着说，"欢迎加入虚空炉火旅店！"
</maintext>

<sum>
时间：第1天 下午15:30 | 地点：虚空炉火旅店大厅 | 人物：店主、新员工艾莉丝 | 事件：精灵法师艾莉丝成功入职成为前台接待

资源.金币: Y

_.set('时间.当前时间[0]', '14:00', '15:30');//剧情发展，时间流逝1小时30分钟

```

## 注意事项

1. **字数限制**：maintext必须严格控制在500-1300字之间
2. **格式要求**：必须包含三个标签，不能缺少任何部分
3. **变量更新**：只更新实际发生变化的变量，不要无中生有
4. **逻辑一致性**：剧情要符合游戏世界观和当前状态
5. **路径格式**：所有变量路径必须加上[0]后缀
