# 开场页面（Splash Screen）需求文档

## 1. 项目概述

### 1.1 项目名称
**诸天便携旅店 - 开场页面**

### 1.2 项目定位
开场页面是玩家进入游戏时看到的第一个界面，需要营造沉浸式的奇幻旅店氛围，引导玩家进入游戏世界。

---

## 1.5 游戏主题与世界观

### 1.5.1 核心主题
**诸天便携旅店** - 一个可以在不同世界观之间自由穿越的神秘旅店。

### 1.5.2 游戏世界观

#### 背景设定
在无尽的多元宇宙中，存在着一家特殊的旅店——**虚空炉火旅店**。这家旅店拥有独特的能力：它可以在不同的世界观之间自由穿越，为来自各个维度的迷失者提供庇护。

旅店本身并没有什么了不起的设施，但它拥有一个独特的能力：**可以自由切换旅馆内部的法则**。这使得旅店可以适应各种不同的客人需求，无论是普通人类、魔法生物，还是来自未来世界的赛博朋克居民。

#### 核心特色

1. **跨世界观穿越**
   - 旅店可以在不同的世界观之间穿越（现代都市、魔法学院、赛博朋克、古代王朝等）
   - 每个世界观都有独特的客人、剧情和挑战
   - 玩家可以通过"当前世界"功能选择想要穿越的世界

2. **旅馆法则系统**
   - 旅店内部可以切换不同的法则，影响客人的行为和剧情发展
   - **禁魔法则**: 禁止使用魔法，适合普通客人
   - **高魔法则**: 高浓度魔法环境，适合魔法生物
   - **发情力场**: 特殊力场，影响客人情绪（支持色色玩法）
   - **好朋友法则**: 强制友好相处，仇人也会和好
   - **中立法则**: 平衡的魔法环境
   - **自由法则**: 无限制，完全自由

3. **旅店经营**
   - 管理员工团队（不同种族、职业、等级的员工）
   - 接待来自各个世界的客人
   - 扩建旅店设施（客房、餐饮、调酒、温泉等）
   - 通过故事系统推进剧情

4. **自由度体现**
   - 玩家可以自定义旅店名称（体现自由度）
   - 可以切换营业状态（营业中/升级中/筹备中），影响剧情
   - 可以管理员工、设施、法则，创造独特的游戏体验

### 1.5.3 旅店设定

#### 旅店名称
- **主名称**: "虚空炉火旅店"（可自定义，体现自由度）
- **副标题**: "迷失者的庇护所"
- **英文名**: "The Ethereal Hearth"

#### 旅店特色
- **温暖氛围**: 炉火温暖地噼啪作响，营造温馨的庇护所氛围
- **神秘感**: 空气中弥漫着烤肉的香味和臭氧的气息，暗示旅店的神秘力量
- **包容性**: 欢迎来自各个维度的疲惫旅人，无论他们来自哪个世界

### 1.5.4 视觉主题

#### 整体风格
- **奇幻**: 魔法与现实的交织
- **温暖**: 金色炉火、温暖光线
- **神秘**: 紫色魔法光效、暗色背景
- **包容**: 多元化的视觉元素，体现跨世界特色

#### 色彩语言
- **主色（金色）**: 代表温暖、炉火、庇护
- **辅助色（紫色）**: 代表魔法、神秘、跨维度
- **背景（深蓝/黑）**: 代表虚空、未知、多元宇宙
- **高光（青色/绿色）**: 代表希望、生机、新世界

### 1.5.5 开场页面主题体现

开场页面需要体现以下核心主题：

1. **跨世界特色**
   - 背景可以展示旅店在不同世界中的样貌（可选：多个世界场景的切换或融合）
   - 视觉元素暗示旅店的穿越能力

2. **温暖庇护所**
   - 强调"迷失者的庇护所"这一核心概念
   - 营造温馨、安全的氛围
   - 金色炉火作为视觉焦点

3. **神秘力量**
   - 暗示旅店的独特能力（法则切换、世界穿越）
   - 使用魔法光效、粒子效果等增强神秘感

4. **自由选择**
   - 开场页面本身就是玩家进入游戏世界的"第一扇门"
   - 设计要体现玩家即将开始一段自由探索的旅程

### 1.5.7 背景故事（用于开场动画或描述）

- **语言**: TypeScript
- **DOM操作**: jQuery（禁止使用原生 `DOMContentLoaded`）

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 主菜单按钮
必须包含以下按钮（按优先级排序）：

1. **开始游戏** (优先级最高)
   - 功能：创建新游戏，跳转到游戏主界面
   - 交互：点击后执行回调 `onStartNewGame()`
   - 状态：始终可见，默认焦点

2. **继续游戏** (条件显示)
   - 功能：加载已保存的游戏进度
   - 交互：点击后执行回调 `onContinueGame()`
   - 状态：仅当存在存档时显示
   - 显示条件：检查是否有存档数据（通过回调参数 `hasSaveData: boolean` 控制）

3. **设置** (可选)
   - 功能：打开设置菜单
   - 交互：点击后执行回调 `onOpenSettings()`
   - 状态：始终可见

4. **退出** (可选)
   - 功能：关闭游戏/返回上级页面
   - 交互：点击后执行回调 `onExit()`
   - 状态：始终可见

#### 2.1.2 游戏标题显示
- **主标题**: "虚空炉火旅店" 或 "诸天便携旅店"（可配置，体现游戏核心主题）
- **副标题**: "迷失者的庇护所" 或自定义副标题（可配置，体现游戏世界观）
- **显示方式**: 大号字体，居中显示，带渐变效果和发光动画
- **主题体现**: 标题设计要体现"跨世界"和"温暖庇护所"的双重主题

#### 2.1.3 背景与氛围
- **背景图片**: 使用与游戏主题一致的旅店场景图片
  - **推荐主题**: 温暖的炉火、木制家具、魔法光效、跨世界元素（可选：多个世界的融合场景）
  - **核心元素**: 必须包含炉火（体现"炉火旅店"主题）
  - **可选元素**: 魔法符文、传送门、多元宇宙的视觉暗示
- **背景效果**: 
  - 模糊效果（backdrop-filter）
  - 渐变遮罩层（深色，确保文字可读性）
  - 可选的粒子效果或光效动画（魔法光点、炉火闪烁等）
  - 可选的背景动画：缓慢的世界切换效果（暗示跨世界能力）
- **氛围音效**: （可选，不在本次实现范围）
- **主题体现**: 背景要同时体现"温暖庇护所"和"神秘跨世界"两个核心主题

### 2.2 交互需求

#### 2.2.1 按钮交互
- **悬停效果**: 
  - 按钮放大（scale: 1.05）
  - 边框发光
  - 颜色渐变变化
  - 过渡动画时长：0.3s ease
- **点击效果**:
  - 按钮按下动画（scale: 0.95）
  - 点击后立即恢复
- **焦点状态**: 
  - 键盘导航支持（Tab键切换）
  - 焦点按钮高亮显示

#### 2.2.2 页面动画
- **进入动画**:
  - 淡入效果（fadeIn）：0.6s ease
  - 标题从上方滑入：translateY(-20px) → translateY(0)
  - 按钮从下方滑入：translateY(20px) → translateY(0)
  - 背景图片缩放进入：scale(1.1) → scale(1)
- **退出动画**:
  - 淡出效果（fadeOut）：0.4s ease
  - 执行退出动画后调用回调函数

#### 2.2.3 加载状态
- **初始加载**: 显示加载动画（可选）
- **数据加载**: 如果检查存档，显示加载提示

```
│          [背景图片/视频]            │

│         └─────────────┘            │
│         ┌─────────────┐            │
```

#### 3.1.2 响应式要求
- **桌面端**: 最大宽度 1920px，居中显示
- **移动端**: 全宽显示，按钮和文字适当缩小
- **高度**: 使用 `width` 和 `aspect-ratio` 控制高度（禁止使用 `vh` 单位）

#### 3.1.3 间距规范
- 使用 `variables.scss` 中定义的间距变量：
  - `$spacing-xs: 4px`
  - `$spacing-sm: 8px`
  - `$spacing-md: 16px`
  - `$spacing-lg: 24px`
  - `$spacing-xl: 32px`
  - `$spacing-xxl: 48px`

### 3.2 视觉设计

#### 3.2.1 颜色方案
必须使用 `variables.scss` 中定义的颜色：

**主色调**:
- 主色（金色）: `$color-primary: #fbbf24` (amber-400)
- 主色浅: `$color-primary-light: #fcd34d` (amber-300)
- 主色深: `$color-primary-dark: #d97706` (amber-600)

**辅助色**:
- 紫色: `$color-secondary: #8b5cf6` (purple-500)
- 青色: `$color-accent-cyan: #06b6d4` (cyan-500)
- 绿色: `$color-accent-emerald: #10b981` (emerald-500)

**背景色**:
- 深色背景: `$color-bg-deep: #0f172a` (slate-950)
- 背景遮罩: `$color-bg-overlay: rgba(15, 23, 42, 0.95)`

**文字色**:
- 主文字: `$color-text-main: #e2e8f0` (slate-200)
- 次要文字: `$color-text-muted: #94a3b8` (slate-400)
- 暗淡文字: `$color-text-dim: #64748b` (slate-500)
- 高亮文字: `$color-text-highlight: #ffffff`

#### 3.2.2 字体规范
必须使用 `variables.scss` 中定义的字体：

- **标题字体**: `$font-title: 'Ma Shan Zheng', cursive` (马善政体)
- **正文字体**: `$font-body: 'Outfit', sans-serif`
- **衬线字体**: `$font-serif: 'Noto Serif SC', serif` (思源宋体)
- **奇幻字体**: `$font-fantasy: 'Ma Shan Zheng', cursive`

**字体大小规范**:
- 主标题: 48px - 64px（桌面端），32px - 40px（移动端）
- 副标题: 18px - 24px（桌面端），14px - 18px（移动端）
- 按钮文字: 16px - 18px（桌面端），14px（移动端）

#### 3.2.3 按钮设计

**按钮样式**:
- 使用 `glass-panel` mixin（毛玻璃效果）
- 圆角: `$radius-lg: 16px`
- 边框: `1px solid rgba(255, 255, 255, 0.1)`
- 内边距: `$spacing-md $spacing-xl` (垂直 16px，水平 32px)
- 最小宽度: 200px（桌面端），160px（移动端）

**按钮颜色**:
- **开始游戏**: 金色渐变 (`$gradient-primary`)
- **继续游戏**: 紫色渐变 (`$gradient-purple`)
- **设置**: 灰色半透明背景
- **退出**: 红色半透明背景（可选）

**按钮阴影**:
- 默认: `$shadow-md: 0 8px 24px rgba(0, 0, 0, 0.4)`
- 悬停: `$shadow-lg: 0 16px 48px rgba(0, 0, 0, 0.6)`
- 发光效果: `$shadow-glow-gold` 或 `$shadow-glow-purple`

#### 3.2.4 背景设计

**背景图片要求**:
- 分辨率: 至少 1920x1080
- 主题: 奇幻旅店场景，必须体现以下元素：
  - **核心元素（必须）**: 温暖的炉火（体现"炉火旅店"主题）
  - **基础元素**: 木制家具、旅店内部场景
  - **魔法元素**: 魔法光效、符文、魔法物品（体现跨世界能力）
  - **可选元素**: 
    - 多个世界的融合场景（现代+奇幻+未来等）
    - 传送门或空间裂缝（暗示跨世界能力）
    - 来自不同世界的客人剪影
- 格式: 支持 JPG/PNG/WebP
- **主题一致性**: 背景图片必须同时体现"温暖庇护所"和"跨世界神秘"两个核心主题

**背景效果**:
- 模糊: `backdrop-filter: blur(8px)`
- 遮罩: 深色渐变遮罩层，确保文字可读性
- 动画: 可选的缓慢缩放或平移动画

## 4. 美学要求

### 4.1 整体风格
- **主题**: 奇幻、温暖、神秘、跨世界
- **氛围**: 营造"迷失者的庇护所"的温馨感，同时暗示旅店的神秘力量
- **调性**: 暗色调背景（代表虚空、多元宇宙）+ 金色高光（代表炉火、温暖）+ 紫色光效（代表魔法、跨维度），形成强烈对比
- **核心情感**: 
  - **温暖**: 金色炉火、温馨光线、庇护所的安全感
  - **神秘**: 紫色魔法、跨世界暗示、未知的可能性
  - **自由**: 多元化的视觉元素、开放的选择感

### 4.2 视觉层次
1. **第一层（背景）**: 旅店场景图片，模糊处理
   - 体现跨世界主题：可以是多个世界的融合场景，或单一世界的代表性场景
   - 核心元素：炉火（必须可见，体现"炉火旅店"）
2. **第二层（遮罩）**: 深色渐变遮罩，增强对比度
   - 可选：遮罩中可以融入魔法光效、粒子效果
3. **第三层（标题）**: 大号标题，金色渐变，发光效果
   - 金色代表炉火和温暖
   - 可选：标题周围添加紫色光晕，暗示魔法力量
4. **第四层（按钮）**: 毛玻璃按钮，悬停时高亮
   - "开始游戏"按钮使用金色渐变（呼应炉火主题）
   - "继续游戏"按钮使用紫色渐变（呼应魔法主题）

### 4.3 动画要求

#### 4.3.1 进入动画序列
1. **背景淡入** (0.6s): `opacity: 0 → 1`
2. **标题滑入** (0.8s, delay: 0.2s): `translateY(-30px) → translateY(0)`, `opacity: 0 → 1`
3. **按钮依次滑入** (0.6s each, delay: 0.4s, 0.5s, 0.6s, 0.7s): 
   - 从下方滑入: `translateY(30px) → translateY(0)`
   - 淡入: `opacity: 0 → 1`

#### 4.3.2 交互动画
- **按钮悬停**: 
  - 缩放: `scale(1) → scale(1.05)`
  - 上移: `translateY(0) → translateY(-4px)`
  - 发光: 阴影增强
  - 时长: 0.3s ease
- **按钮点击**:
  - 缩放: `scale(1.05) → scale(0.95) → scale(1.05)`
  - 时长: 0.15s ease

#### 4.3.3 背景动画（可选）
- **缓慢缩放**: `scale(1) → scale(1.05)`，循环，时长 20s
- **轻微移动**: 背景图片轻微平移，营造动态感

### 4.4 特效要求

#### 4.4.1 文字特效
- **标题渐变**: 使用 `text-gradient-gold` mixin（金色，代表炉火）
- **标题发光**: 
  - 金色发光: `text-shadow: 0 0 20px rgba(251, 191, 36, 0.5)`
  - 可选紫色光晕: `text-shadow: 0 0 30px rgba(139, 92, 246, 0.3)`（暗示魔法力量）
- **副标题**: 较小的发光效果，颜色较淡，可以使用紫色调（呼应"迷失者的庇护所"的神秘感）
- **主题体现**: 文字特效要同时体现温暖（金色）和神秘（紫色）两个主题

#### 4.4.2 按钮特效
- **毛玻璃效果**: 使用 `glass-panel` mixin
- **边框发光**: 悬停时边框颜色变为 `$color-border-hover`
- **渐变背景**: 主按钮使用渐变背景

### 5.2 组件接口定义

```typescript
export interface SplashScreenCallbacks {
  onStartNewGame: () => void;
  onContinueGame?: () => void;
  onOpenSettings?: () => void;
  onExit?: () => void;
}

export interface SplashScreenOptions {
  title?: string;                    // 主标题，默认: "虚空炉火旅店"
```

## 6. 交互流程

### 6.1 页面加载流程
1. 页面加载 → 显示加载动画（可选）
2. 检查存档数据 → 设置 `hasSaveData` 标志
3. 渲染开场页面 → 调用 `show()`
4. 执行进入动画 → 依次显示各元素
5. 等待用户交互

### 6.2 用户操作流程

#### 6.2.1 开始新游戏
1. 用户点击"开始游戏"按钮
2. 执行退出动画（0.4s）
3. 调用 `onStartNewGame()` 回调
4. 隐藏开场页面
5. 显示游戏主界面

#### 6.2.2 继续游戏
1. 用户点击"继续游戏"按钮（仅当有存档时显示）
2. 显示加载提示
3. 调用 `onContinueGame()` 回调
4. 加载游戏数据
5. 隐藏开场页面
6. 显示游戏主界面

#### 6.2.3 打开设置
1. 用户点击"设置"按钮
2. 调用 `onOpenSettings()` 回调
3. 显示设置菜单（由回调函数处理）

#### 6.2.4 退出游戏
1. 用户点击"退出"按钮
2. 显示确认对话框（可选）
3. 调用 `onExit()` 回调
4. 执行退出操作（由回调函数处理）
