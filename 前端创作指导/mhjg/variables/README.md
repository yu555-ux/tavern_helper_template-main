# 游戏变量系统说明

## 文件说明

- `[InitVar]初始变量.json`: 游戏变量的初始值定义文件，符合 MVU (MagVarUpdate) 系统规范

## 变量结构说明

### 变量格式

每个变量都遵循以下格式：
```json5
"变量名": ["初始值", "更新条件和描述"]
```

### 变量分类

#### 1. 旅店信息 (`旅店`)
- **名称**: 旅店名称，开局时设定
- **当前世界**: 对象，包含：
  - `世界名`: 旅店所在的世界名称，可通过世界旅行切换
  - `世界介绍`: 当前世界的详细描述和特点
- **营业状态**: 营业中/升级中/筹备中
- **旅馆法则**: 对象，包含：
  - `法则列表`: 数组，包含所有持有的法则。每个法则是对象，包含：
    - `法则名`: 法则名称（禁魔、高魔、发情力场、好朋友法则、中立、自由）
    - `法则效果`: 法则的详细效果描述
- **受欢迎度**: 0-100，反映客人的喜爱程度
- **入住率**: 0-100，反映客房使用情况

#### 2. 资源 (`资源`)
- **金币**: 主要货币，用于各种消费
- **声望**: 旅店的名气和影响力
- **建材**: 用于扩建和升级的材料

#### 3. 时间 (`时间`)
- **天数**: 游戏开始后的天数
- **当前时间**: 一天中的具体时间

#### 4. 设施 (`设施`)
- **设施列表**: 数组，包含所有设施。每个设施是对象，包含：
  - `名称`: 设施名称（如客房、餐饮、调酒、温泉等）
  - `等级`: 0-5级（0为未建造，1-5为不同等级）
  - `当前等级描述`: 对应等级的文字描述

#### 5. 店主 (`店主`)
- **身份**: 开局时选择的身份类型
- **初始福利**: 开局时选择的特殊福利

#### 6. 客人 (`客人`)
- **客人列表**: 数组，包含所有当前在店的客人。每个客人是对象，包含：
  - `名称`: 客人姓名
  - `种族`: 如人类、精灵、矮人等
  - `职业名`: 如流浪骑士、巡林客、铁匠等
  - `职业等级`: 整数，代表职业等级
  - `外貌描述`: 客人的外貌特征描述
  - `心情值`: 0-100，代表客人当前的心情
  - `消费金额`: 整数，客人本次消费的金币数
  - `当前想法`: 客人当前的想法或需求描述

#### 7. 员工 (`员工`)
- **员工列表**: 数组，包含所有雇佣的员工。每个员工是对象，包含：
  - `名称`: 员工姓名
  - `种族`: 如精灵、矮人、暗精灵等
  - `战斗职业名`: 如法师、战士、盗贼等
  - `职业等级`: 整数，代表战斗职业等级
  - `旅馆担任的职务`: 如前台接待、厨师、保安等
  - `外貌描述`: 员工的外貌特征描述
  - `对玩家的好感度`: 0-100，代表员工对店主的好感
  - `工作满意度`: 0-100，代表员工对今天工作环境的满意度
  - `喜好`: 字符串数组，包含员工喜欢的事物（可以有多个标签）
  - `厌恶`: 字符串数组，包含员工厌恶的事物（可以有多个标签）
  - `工资`: 整数，员工每月的工资

#### 8. 特殊事件 (`特殊事件`)
- **当前事件**: 正在进行的特殊事件
- **事件进度**: 事件完成进度

## 使用方式

### 在角色卡中使用

1. 将 `[InitVar]初始变量.json` 的内容复制到角色卡的世界书条目中
2. 条目名称必须包含 `[InitVar]` 或 `[initvar]`（不区分大小写）
3. 条目可以是关闭状态，MVU 系统会自动读取

### 在提示词模板中使用

```ejs
<% if (_.has(getvar("stat_data"), '旅店.名称.[0]')) { %>
旅店名称: <%- getvar("stat_data").旅店.名称[0] %>
<% } %>
```

### 变量更新格式

LLM 需要按照以下格式输出变量更新：

#### 简单变量更新
```
<UpdateVariable>
<Analysis>
旅店.受欢迎度: Y
资源.金币: Y
</Analysis>
_.set('旅店.受欢迎度', 50, 55);//客人对服务很满意
_.set('资源.金币', 500, 650);//客人消费了150金币
</UpdateVariable>
```

#### 数组变量更新

对于数组类型的变量（如法则列表、设施列表、客人列表、员工列表），需要更新整个数组：

```
<UpdateVariable>
<Analysis>
旅馆法则.法则列表: Y
</Analysis>
_.set('旅馆法则.法则列表', [旧数组], [新数组]);//获得新法则"高魔"
</UpdateVariable>
```

**注意**: 根据 MVU 系统规范，数组变量更新时只更新第一个元素，不输出 `[]` 块。但在实际使用中，如果需要添加新元素，需要更新整个数组。

## 注意事项

1. **数组访问**: 所有变量值都是数组格式 `[值, "描述"]`，访问时需要使用 `[0]` 获取实际值
2. **变量路径**: 使用点号分隔的路径，如 `旅店.名称`、`资源.金币`
3. **更新条件**: 每个变量的描述部分会指导 LLM 何时更新该变量
4. **初始值覆盖**: 可以在额外问候语中使用 `<UpdateVariable>` 段来覆盖初始值
